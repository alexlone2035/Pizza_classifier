1. Функционал модулей

1.1. Telegram-бот (Пользовательский интерфейс)
   
Прием фотографий пиццы от пользователей.
Выдача инструкций (например, "сделайте фото под углом 45 градусов для проверки слоев").

Отправка результатов проверки:

Распознанный тип пиццы.

Вердикт (Всё ОК / Найдены отклонения).

Картинка с визуализацией (отрисованные bounding boxes вокруг проблемных зон или ингредиентов).

Сбор обратной связи от пользователя (кнопки "Модель права" / "Модель ошиблась").

1.2. Backend & API (Связующее звено)

Серверная логика, принимающая запросы от Telegram-бота.

Интеграция с ML-модулем (отправка фото на инференс, получение JSON с результатами).

Логирование истории проверок в базу данных.

1.3. ML-модуль (Аналитика)

Классификатор: Определение типа пиццы.

Детектор объектов (Instance Segmentation): Поиск пиццы.

1.4. Данные и Админка

База собранных фотографий "правильных" и "неправильных" пицц.

Простой дашборд для просмотра статистики работы бота (количество проверок, процент брака).

2. Распределение задач (Команда)
   
2.1. Сбор данных, разметка и тестирование (Умерзакова Камилла)

Без программирования:

Сбор сырого датасета (парсинг картинок, поиск фото пицц разных видов, включая бракованные).

Ручная разметка данных в Roboflow/CVAT.

Создание синтетического "брака" (если реальных фото брака не хватает).

Ручное тестирование бота (QA): закидывать фото с телефона, ломать сценарии использования, проверять адекватность ответов.

Ведение документации и подготовка презентации проекта.

2.2. Machine Learning: классификация и детекция (Семикин Алексей)

Выбор и обучение архитектуры для классификации видов пиццы.

Настройка и обучение модели для детекции ингредиентов и сегментации масок.

Написание логики оценки "Всё ли ок" на основе выхода нейросетей (подсчет боксов, анализ слоев).

Упаковка модели в инференс-скрипт.

2.3. Backend: логика и интеграция (Крюков Сергей)

Разработка API (FastAPI) для приема фото от бота и передачи их в ML-модель.

Проектирование и создание базы данных (сохранение логов проверок и фидбека).

Настройка стабильного взаимодействия между сервером и ботом.

Отрисовка результатов (наложение Bounding Boxes на фото для отправки пользователю).

2.4. UX, Telegram-бот и DevOps (Миргазиев Эмиль)

Написание самого Telegram-бота (aiogram/telebot).

Проектирование UX-сценариев общения бота с пользователем (обработка ошибок, неверных форматов файлов).

Создание простого дашборда со статистикой (например, на Streamlit).

Деплой проекта (упаковка в Docker, запуск на сервере).


3. Деплой проекта

   В корне проекта создать файл .env, содержащий:
```
API_KEY="Ваш ключ"
TELEGRAM_BOT_TOKEN="Ваш токен"
API_BASE_URL=http://api:8000
POSTGRES_USER="Пользователь БД"
POSTGRES_PASSWORD="Пароль от БД"
POSTGRES_DB="Название БД"
```

   Для запуска проекта на docker необходимо из корня проекта ввести консольную команду `docker-compose up --build`
